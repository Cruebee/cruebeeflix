Documentation

This application will provide users with access to information about different movies, directors, and genres.
Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Request URL HTTP Method Request Body Data Format Response Body Data Format
Get a list of ALL movies /movies GET None A JSON object holding data about all movies with the following format:

Example: { "Genre": [ "5162df6df5a1d6f5sdf" ], "Director": [ "56656651658df651asdf8" ], "_id": "562s1df651s6d2a65df9", "Title": "The Boondock Saints", "Description": "The Saints are not going to sit around while crime runs ramptant in their city.", "ImagePath": "boondocksaints.svg", "Featured": true }

Get data about a single movie by its title /movies/:Title GET None A JSON object containing information about a single movie, including the title, author, screenwriter, year, description, genre, and director.

Example: { "Genre": [ "55ad6f51df651sd651" ], "Director": [ "565sdf169s51f9a8s5df" ], "_id": "5sd65sdf54656df1sdf9", "Title": "The Boondock Saints", "Description": "The Saints are not going to sit around while crime runs ramptant in their city.", "ImagePath": "boondocksaints.jpg", "Featured: true" }

Get data about ALL genres /genres GET None A list of JSON objects holding genre names, and a description about the genre.

Example: { "_id": "5sdfs65d1fas65f1a6s51", "Name": "Comedy", "Description": "Comedy films are associated with the type of work that is amusing and satirical. Comedies are mostly having a cheerful ending. Tragedy is the opposite to a comedy, as tragedy deals with sorrowful and tragic events in a story." }

Get data about a specific genre by name /genres/:Name GET None A JSON object holding a genre name and a description about the genre:

Example: { "_id": "5df621df65we165fg65", "Name": "Thriller", "Description": "Thriller is a broad genre." }

Get data about all directors /directors GET None A list of JSON objects holding information about directors each of which will be in the following format:

Example: { "_id": "Name": "George Lucas", "Bio": "George is best known for creating the 'Star Wars' and 'Indiana Jones' franchises.", "Birth": "1944-05-14", "Death": "(if applicable)" }

Get data about a director by name /directors/:Name GET None A JSON object holding information about a director in the following format:

Example: { "_id": "5df521651fa65df6s51f85", "Name": "George Lucas", "Bio": "George Lucas is best known for creating the 'Star Wars' and 'Indiana Jones' franchises.", "Birth": "1944-05-14", "Death": "(if applicable)" }

Add new user /users POST A JSON object holding data about the user being added, with the following format:

Example: { "Username": "johnnydoe23", "Password": "userpassword123", "Email": "example@email.com", "Birthday": "1989-08-12" }

A JSON object holding the data about the new user, including a new ID and an empty FavoriteMovies array:

Example: { "FavoriteMovies" : [], "_id": "5d50550d054dga505ssdf", "Username": "johnnydoe23", "Password": "userPassword123", "Email": "emailexample@email.com", "Birthday": "1989-11-12", "__v": 0 }

Allow users to update their information by Username /users/:Username PUT A JSON object holding the data to be updated with the format:

Example: { "FavoriteMovies": [ { "5465d2fs65fa6s5dfdsf5" } ], "_id": '5d15d6ad51gas6d5sdf4', "Username": "updated username", "Email": "updated email", "Birthday": "updated birthday", "__v": 0 }

A JSON object containing the updated information.
Allow users to add movies to a list of favorites using their ID to add a movie by MovieID /users/:Username/movies/:MovieID POST None A message indicating the movie has been added to favorites
Allow users to remove movies from favorites list /users/:Username/movies/:MovieID DELETE A JSON object containing the updated information:

Example: { "FavoriteMovies": [ "655d55958d21f6sd5f1a6", "450sdf206sdf2asdf06sd" ], "_id": "65df65df5sd98sd9f8ss3", "Username": "bobber", "Password": "password", "Email": "email@email.com", "Birthday": "1999-11-11T00:00:00:.00z", "__v": 0 }

Allow users to delete their account by Username /users/:Username DELETE A message indicating the user account has been removed
